# Sucana Grok X Monitoring

**Know what your audience is complaining about before they tell you.**

## The Problem

Your audience is on X right now, talking about the exact problems your product solves. They're venting about tools that don't work, asking questions nobody answers, sharing workarounds that shouldn't exist.

That's gold for content, product decisions, and positioning. But you'd have to spend hours scrolling X every week to find it. And by the time you do, the moment's passed.

## What This Skill Does

Grok X Monitoring is a Claude Code skill that scans X/Twitter weekly using the Grok API (xAI). It searches for specific pain clusters your audience talks about, pulls real posts, and organizes everything into a structured analysis.

What you get every week:
- **Pain points** people are actually expressing (with real quotes)
- **Trending topics** in your space
- **Content hooks** you can turn into posts, articles, or ads
- **Shifts** from last week's scan (what's growing, what's fading)

## How It Works

1. You define your pain clusters (the problems your audience has)
2. The skill calls Grok's API with X search enabled
3. It scans across all your clusters in one run
4. It outputs a structured analysis with real posts and patterns
5. New hooks get added to your content ideas file

One command. Six pain clusters scanned. Weekly research done in 2 minutes instead of 2 hours.

## Install

Drop the `grok-x-monitoring/` folder into your `.claude/skills/` directory.

## Requirements

- Claude Code CLI
- `XAI_API_KEY` in your `.env` file (get one at console.x.ai — $25 free credits on signup)
